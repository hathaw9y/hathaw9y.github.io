## Skewing이란?

앞서 OS/WS 구현에서 모든 행의 `act_i`가 동시에 입력되면 타이밍이 어긋나는 문제를 확인했다.

Systolic Array에서 각 PE는 입력 데이터를 매 cycle마다 이웃 PE로 전달한다.
따라서 데이터는 배열 안에서 한 칸 이동할 때마다 1 cycle씩 늦어진다.

문제는 GEMM에서 같은 `k`에 해당하는 activation과 weight가 같은 PE에서 같은 cycle에 만나야 한다는 점이다.

$$C_{i,j} = \sum_k A_{i,k} \times B_{k,j}$$

예를 들어 어떤 PE가 `A[i][k]`와 `B[k][j]`를 곱해야 한다면, 두 값은 같은 cycle에 그 PE에 도착해야 한다.
하지만 activation은 왼쪽에서 오른쪽으로 이동하고, weight는 위에서 아래로 이동하므로 PE 위치에 따라 도착 시간이 달라진다.

Skewing은 입력을 미리 지연시켜서 이 도착 시간을 맞추는 기법이다.
즉 데이터를 직사각형 모양으로 동시에 넣는 것이 아니라, 대각선 wavefront처럼 밀어 넣는다.

```text
cycle 0: row0 입력
cycle 1: row1 입력
cycle 2: row2 입력
...
```

행 방향 activation은 보통 행 번호만큼 지연한다.

| 행 | 지연 클락 |
|---|---|
| 행 0 | 0 클락 |
| 행 1 | 1 클락 |
| 행 2 | 2 클락 |
| 행 N-1 | N-1 클락 |

OS처럼 weight도 외부에서 매 cycle stream으로 들어오는 구조라면, weight 역시 열 번호만큼 지연해야 한다.

| 열 | 지연 클락 |
|---|---|
| 열 0 | 0 클락 |
| 열 1 | 1 클락 |
| 열 2 | 2 클락 |
| 열 N-1 | N-1 클락 |

이렇게 하면 같은 `k`에 해당하는 activation과 weight가 각 PE에서 같은 cycle에 만나게 된다.

## pipeline_reg 구현

`pipeline_reg`는 입력 데이터를 `DEPTH` 클락만큼 지연시키는 shift register다.
행 번호나 열 번호를 `DEPTH`로 설정하면 입력을 원하는 cycle만큼 늦출 수 있다.

이 모듈은 데이터뿐 아니라 valid나 clear 같은 1-bit control signal에도 그대로 사용할 수 있다.
중요한 점은 **데이터와 그 데이터의 valid를 반드시 같은 cycle만큼 지연해야 한다**는 것이다.
데이터만 지연하고 valid를 지연하지 않으면 PE는 잘못된 cycle에 MAC을 수행한다.

FPGA에서는 `DEPTH`가 곧 register 개수다.
예를 들어 16개 row를 skewing하면 row 0은 delay가 없지만 row 15는 15-stage shift register를 가진다.
데이터 폭이 넓거나 배열이 커질수록 skewing register도 resource와 timing에 영향을 준다.
그래도 이 지연을 명시적으로 넣어두면 data와 valid의 cycle 관계를 RTL에서 확인하기 쉬워진다.

### 포트 설명

| 포트 | 방향 | 비트폭 | 설명 |
|---|---|---|---|
| `aclk_i` | input | 1-bit | 클락 |
| `aresetn_i` | input | 1-bit | 액티브 로우 리셋 |
| `data_i` | input | DATA_W | 입력 데이터 |
| `data_o` | output | DATA_W | DEPTH 클락 지연된 출력 |

### 코드 구현

```verilog
module pipeline_reg #(
    parameter int DATA_W = 8,  // 데이터 비트폭
    parameter int DEPTH  = 0   // 지연 클락 수 (행 번호와 동일)
) (
    input  logic              aclk_i,
    input  logic              aresetn_i,
    input  logic [DATA_W-1:0] data_i,
    output logic [DATA_W-1:0] data_o
);

  if (DEPTH == 0) begin : g_no_delay
    assign data_o = data_i;
  end else begin : g_delay
    logic [DATA_W-1:0] shift_reg[DEPTH];

    always_ff @(posedge aclk_i) begin
      if (!aresetn_i) begin
        for (int i = 0; i < DEPTH; i++) shift_reg[i] <= '0;
      end else begin
        shift_reg[0] <= data_i;
        for (int i = 1; i < DEPTH; i++) shift_reg[i] <= shift_reg[i-1];
      end
    end

    assign data_o = shift_reg[DEPTH-1];
  end

endmodule
```


## OS+SKEW

### Skewing 동작

OS에서는 activation과 weight가 모두 compute 단계에서 계속 stream된다.
따라서 두 입력 모두 skewing해야 한다.

- `act_i[row]`: `row` 클락만큼 지연 후 해당 행 왼쪽 끝 PE로 입력
- `weight_i[col]`: `col` 클락만큼 지연 후 해당 열 위쪽 끝 PE로 입력
- `act_valid_i`, `weight_valid_i`, `acc_clear_i`도 동일하게 지연

여기서 `acc_clear_i`도 activation과 같은 row 기준으로 지연한다.
OS에서는 accumulator가 PE 내부에 고정되어 있으므로, tile이 바뀔 때 각 PE의 accumulator를 데이터 흐름과 같은 timing으로 clear해야 한다.

```text
act row0: delay 0
act row1: delay 1
act row2: delay 2

weight col0: delay 0
weight col1: delay 1
weight col2: delay 2
```

즉 OS skewing의 목적은 두 가지다.

| 대상 | 이유 |
|---|---|
| activation skew | 행 위치에 따른 오른쪽 이동 latency 보정 |
| weight skew | 열 위치에 따른 아래쪽 이동 latency 보정 |
| clear skew | 각 PE accumulator 초기화 timing 정렬 |

### systolic_array_os.sv

```verilog
module systolic_array_os #(
    parameter int ROWS     = 16,
    parameter int COLS     = 16,
    parameter int ACT_W    = 8,
    parameter int WEIGHT_W = 8,
    parameter int ACC_W    = 32
) (
    input  logic                       aclk_i,
    input  logic                       aresetn_i,
    input  logic signed [   ACT_W-1:0] act_i         [ROWS],
    input  logic signed [WEIGHT_W-1:0] weight_i      [COLS],
    input  logic                       act_valid_i   [ROWS],
    input  logic                       weight_valid_i[COLS],
    input  logic                       acc_clear_i,
    output logic signed [   ACC_W-1:0] acc_o         [ROWS][COLS]
);

  // Skewed 신호
  logic signed [   ACT_W-1:0] act_skewed         [  ROWS];
  logic                       act_valid_skewed   [  ROWS];
  logic                       acc_clear_skewed   [  ROWS];
  logic signed [WEIGHT_W-1:0] weight_skewed      [  COLS];
  logic                       weight_valid_skewed[  COLS];
  // PE 간 연결 wire
  logic signed [   ACT_W-1:0] act_wire           [  ROWS] [COLS+1];
  logic signed [WEIGHT_W-1:0] weight_wire        [ROWS+1] [  COLS];
  logic                       act_valid_wire     [  ROWS] [COLS+1];
  logic                       weight_valid_wire  [ROWS+1] [  COLS];
  logic                       acc_clear_wire     [  ROWS] [COLS+1];

  // =========================================================
  // Skewing: act는 행 번호, weight는 열 번호만큼 지연
  // =========================================================
  genvar i;
  generate
    // act skewing (행 방향)
    for (i = 0; i < ROWS; i++) begin : g_skew_act
      pipeline_reg #(
          .DATA_W(ACT_W),
          .DEPTH (i)
      ) u_skew_act (
          .aclk_i   (aclk_i),
          .aresetn_i(aresetn_i),
          .data_i   (act_i[i]),
          .data_o   (act_skewed[i])
      );

      pipeline_reg #(
          .DATA_W(1),
          .DEPTH (i)
      ) u_skew_act_valid (
          .aclk_i   (aclk_i),
          .aresetn_i(aresetn_i),
          .data_i   (act_valid_i[i]),
          .data_o   (act_valid_skewed[i])
      );

      pipeline_reg #(
          .DATA_W(1),
          .DEPTH (i)
      ) u_skew_clear (
          .aclk_i   (aclk_i),
          .aresetn_i(aresetn_i),
          .data_i   (acc_clear_i),
          .data_o   (acc_clear_skewed[i])
      );
    end

    // weight skewing (열 방향)
    for (i = 0; i < COLS; i++) begin : g_skew_weight
      pipeline_reg #(
          .DATA_W(WEIGHT_W),
          .DEPTH (i)
      ) u_skew_weight (
          .aclk_i   (aclk_i),
          .aresetn_i(aresetn_i),
          .data_i   (weight_i[i]),
          .data_o   (weight_skewed[i])
      );

      pipeline_reg #(
          .DATA_W(1),
          .DEPTH (i)
      ) u_skew_weight_valid (
          .aclk_i   (aclk_i),
          .aresetn_i(aresetn_i),
          .data_i   (weight_valid_i[i]),
          .data_o   (weight_valid_skewed[i])
      );
    end
  endgenerate

  // =========================================================
  // 입력 연결
  // =========================================================
  generate
    for (i = 0; i < ROWS; i++) begin : g_input_act
      assign act_wire[i][0]       = act_skewed[i];
      assign act_valid_wire[i][0] = act_valid_skewed[i];
      assign acc_clear_wire[i][0] = acc_clear_skewed[i];
    end
    for (i = 0; i < COLS; i++) begin : g_input_weight
      assign weight_wire[0][i]       = weight_skewed[i];
      assign weight_valid_wire[0][i] = weight_valid_skewed[i];
    end
  endgenerate

  // =========================================================
  // PE 배열
  // =========================================================
  genvar row, col;
  generate
    for (row = 0; row < ROWS; row++) begin : g_row
      for (col = 0; col < COLS; col++) begin : g_col
        pe_os #(
            .ACT_W   (ACT_W),
            .WEIGHT_W(WEIGHT_W),
            .ACC_W   (ACC_W)
        ) u_pe (
            .aclk_i        (aclk_i),
            .aresetn_i     (aresetn_i),
            .act_valid_i   (act_valid_wire[row][col]),
            .weight_valid_i(weight_valid_wire[row][col]),
            .acc_clear_i   (acc_clear_wire[row][col]),
            .act_i         (act_wire[row][col]),
            .weight_i      (weight_wire[row][col]),
            .act_valid_o   (act_valid_wire[row][col+1]),
            .weight_valid_o(weight_valid_wire[row+1][col]),
            .acc_clear_o   (acc_clear_wire[row][col+1]),
            .act_o         (act_wire[row][col+1]),
            .weight_o      (weight_wire[row+1][col]),
            .acc_o         (acc_o[row][col])
        );
      end
    end
  endgenerate

endmodule
```

## WS+Skewing

### Skewing 동작

WS에서는 weight가 compute 전에 PE 내부에 저장된다.
따라서 compute 단계에서 매 cycle 흘러가는 입력은 activation과 acc이다.

weight load가 끝났다고 가정하면, compute 단계에서는 activation만 row 번호만큼 skewing한다.

- `act_i[row]`: `row` 클락만큼 지연 후 해당 행 왼쪽 끝 PE로 입력
- OS와 달리 weight는 이미 PE 내부에 고정되어 있으므로 compute stream에서는 skew 불필요

다만 weight load 단계 자체는 별도의 timing 제어가 필요하다.
각 PE가 자기 weight를 정확한 cycle에 `weight_reg`에 저장해야 하기 때문이다.
이 글의 코드는 compute path의 skewing에 집중한다.

### Deskewing 동작

WS에서는 partial sum이 위에서 아래로 흐른다.
activation을 row 번호만큼 skew하면 각 column의 결과도 서로 다른 timing으로 출력된다.

즉 배열 맨 아래의 `acc_o[col]`들은 논리적으로 같은 tile의 결과이지만, cycle 관점에서는 정렬되어 있지 않다.
store logic이 한 cycle에 여러 column 결과를 함께 다루려면 output timing을 다시 맞춰야 한다.
이 보정이 deskewing이다.

- 열 0: 가장 먼저 완성
- 열 1: 1 클락 늦게 완성
- 열 N-1: N-1 클락 늦게 완성

이를 보정하기 위해 출력단에서 **역방향 지연**을 추가한다.
빨리 나온 column은 더 오래 지연하고, 늦게 나온 column은 덜 지연한다.

| 열 | deskew 지연 클락 |
|---|---|
| 열 0 | COLS-1 클락 |
| 열 1 | COLS-2 클락 |
| 열 N-1 | 0 클락 |

이렇게 하면 모든 열의 결과가 동일한 클락에 출력된다.

Deskewing도 결국 output 쪽 shift register다.
즉 WS는 input activation을 맞추기 위한 skew register와 output result를 맞추기 위한 deskew register를 모두 가진다.
이 때문에 OS보다 단순 PE는 비슷해 보여도, 주변 timing 정렬 logic은 더 복잡해진다.

WS의 전체 timing 보정은 다음처럼 볼 수 있다.

| 위치 | 보정 |
|---|---|
| 입력 activation | `row`만큼 skew |
| 출력 acc | `COLS - 1 - col`만큼 deskew |

### systolic_array_ws.sv

```verilog
module systolic_array_ws #(
    parameter int ROWS     = 16,
    parameter int COLS     = 16,
    parameter int ACT_W    = 8,
    parameter int WEIGHT_W = 8,
    parameter int ACC_W    = 32
) (
    input  logic                       aclk_i,
    input  logic                       aresetn_i,
    input  logic                       weight_valid_i,
    input  logic                       act_valid_i   [ROWS],
    input  logic signed [   ACT_W-1:0] act_i         [ROWS],
    input  logic signed [WEIGHT_W-1:0] weight_i      [COLS],
    output logic                       acc_valid_o   [COLS],
    output logic signed [   ACC_W-1:0] acc_o         [COLS]
);

  // Skewed 신호
  logic signed [   ACT_W-1:0] act_skewed        [  ROWS];
  logic                       act_valid_skewed  [  ROWS];

  // PE 간 연결 wire
  logic signed [   ACT_W-1:0] act_wire          [  ROWS] [COLS+1];
  logic signed [   ACC_W-1:0] acc_wire          [ROWS+1] [  COLS];
  logic signed [WEIGHT_W-1:0] weight_wire       [ROWS+1] [  COLS];
  logic                       act_valid_wire    [  ROWS] [COLS+1];
  logic                       acc_valid_wire    [ROWS+1] [  COLS];

  // Deskewed 신호
  logic signed [   ACC_W-1:0] acc_deskewed      [  COLS];
  logic                       acc_valid_deskewed[  COLS];

  // =========================================================
  // Skewing: act는 행 번호만큼 지연
  // =========================================================
  genvar i;
  generate
    for (i = 0; i < ROWS; i++) begin : g_skew_act
      pipeline_reg #(
          .DATA_W(ACT_W),
          .DEPTH (i)
      ) u_skew_act (
          .aclk_i   (aclk_i),
          .aresetn_i(aresetn_i),
          .data_i   (act_i[i]),
          .data_o   (act_skewed[i])
      );

      pipeline_reg #(
          .DATA_W(1),
          .DEPTH (i)
      ) u_skew_act_valid (
          .aclk_i   (aclk_i),
          .aresetn_i(aresetn_i),
          .data_i   (act_valid_i[i]),
          .data_o   (act_valid_skewed[i])
      );
    end
  endgenerate

  // =========================================================
  // 입력 연결
  // =========================================================
  generate
    for (i = 0; i < ROWS; i++) begin : g_input_act
      assign act_wire[i][0]       = act_skewed[i];
      assign act_valid_wire[i][0] = act_valid_skewed[i];
    end

    for (i = 0; i < COLS; i++) begin : g_input_acc
      assign acc_wire[0][i]       = '0;
      assign acc_valid_wire[0][i] = 1'b1;

      assign weight_wire[0][i]    = weight_i[i];
    end
  endgenerate

  // =========================================================
  // Deskewing: acc는 (전체 열 - 열 번호)만큼 지연
  // =========================================================
  generate
    for (i = 0; i < COLS; i++) begin : g_output
      pipeline_reg #(
          .DATA_W(ACC_W),
          .DEPTH (COLS - 1 - i)
      ) u_deskew_acc (
          .aclk_i   (aclk_i),
          .aresetn_i(aresetn_i),
          .data_i   (acc_wire[ROWS][i]),
          .data_o   (acc_deskewed[i])
      );

      pipeline_reg #(
          .DATA_W(1),
          .DEPTH (COLS - 1 - i)
      ) u_deskew_acc_valid (
          .aclk_i   (aclk_i),
          .aresetn_i(aresetn_i),
          .data_i   (acc_valid_wire[ROWS][i]),
          .data_o   (acc_valid_deskewed[i])
      );

      assign acc_valid_o[i] = acc_valid_deskewed[i];
      assign acc_o[i]       = acc_deskewed[i];
    end
  endgenerate

  // =========================================================
  // PE 배열
  // =========================================================
  genvar row, col;
  generate
    for (row = 0; row < ROWS; row++) begin : g_row
      for (col = 0; col < COLS; col++) begin : g_col
        pe_ws #(
            .ACT_W   (ACT_W),
            .WEIGHT_W(WEIGHT_W),
            .ACC_W   (ACC_W)
        ) u_pe (
            .aclk_i        (aclk_i),
            .aresetn_i     (aresetn_i),
            .act_valid_i   (act_valid_wire[row][col]),
            .weight_valid_i(weight_valid_i),
            .acc_valid_i   (acc_valid_wire[row][col]),
            .act_i         (act_wire[row][col]),
            .weight_i      (weight_wire[row][col]),
            .acc_i         (acc_wire[row][col]),
            .act_valid_o   (act_valid_wire[row][col+1]),
            .acc_valid_o   (acc_valid_wire[row+1][col]),
            .act_o         (act_wire[row][col+1]),
            .weight_o      (weight_wire[row+1][col]),
            .acc_o         (acc_wire[row+1][col])
        );
      end
    end
  endgenerate
endmodule
```

## 정리

Skewing은 Systolic Array의 기능 추가라기보다, 올바른 GEMM timing을 만들기 위한 입력 정렬이다.

- PE 사이 데이터 이동은 1 cycle latency를 가진다.
- 같은 `k`의 activation과 weight가 같은 PE에서 같은 cycle에 만나야 한다.
- OS는 activation과 weight가 모두 stream되므로 둘 다 skewing한다.
- OS의 `acc_clear_i`도 accumulator 초기화 timing을 맞추기 위해 skewing한다.
- WS는 compute 단계에서 activation을 skewing하고, 출력 acc를 deskewing한다.
- 데이터와 valid/control 신호는 항상 같은 depth로 지연해야 한다.
